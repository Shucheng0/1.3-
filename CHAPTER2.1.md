[数组](https://time.geekbang.org/column/article/40961)：“**一种线性表数据结构，用一组连续的内存空间，来存储一组具有相同类型的数据。**”

那么问题来了，什么是线性表的数据结构呢？使用一组连续的内存空间的意义在那里？为何要限定存储相同类型的数据呢？

| 问题                         | 回答                                                         |
| ---------------------------- | ------------------------------------------------------------ |
| 什么是线性表的数据结构？     | 就是每个线表上的数据最多只有前、后两个方向                   |
| 什么是非线性表的数据结构呢？ | 不是简单的前后关系                                           |
| 使用一组连续内存空间的意义   | 支持根据下标的随机访问                                       |
| 限定存储相同类型的数据       | 数组在定义时，会开始分配内存空间，不同类型数据每个字符占用的存储单元是不同的 |

当数据采用数组形式进行存储时，随机插入或删除的事件复杂度就为O(n)。改进的方案如下：
   * 插入：若存储的数据没有规律，那么可把第 k 位元素移到数组末尾，将新的元素放在第 k 个位置<img src="https://static001.geekbang.org/resource/image/3f/dc/3f70b4ad9069ec568a2caaddc231b7dc.jpg" alt="x 插入第 3 个位置" style="zoom:100%;" />
    * ![依次将 a b c 三个元素删除](https://static001.geekbang.org/resource/image/b6/e5/b69b8c5dbf6248649ddab7d3e7cfd7e5.jpg)
        * 删除：多次删除集中在一起操作。每次删除操作，记录数据已经被删除，再一次出发真正的删除操作。这也是 JVM 标记清除回收算法的核心思想。
